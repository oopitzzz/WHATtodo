## 1. 문서 개요

### 1.1 목적

본 문서는 WHATtodo 애플리케이션의 실제 사용 상황을 구체적으로 묘사하여 다음 목적을 달성하기 위해 작성되었습니다:

- 실제 사용자의 관점에서 제품의 기능과 흐름 이해
- 개발팀과 디자인팀 간 공통된 사용자 경험 비전 공유
- UI/UX 설계 시 고려해야 할 사용자 니즈 파악
- 개발 우선순위 결정을 위한 실사용 맥락 제공
- QA 및 테스트 시나리오 기반 자료 제공

### 1.2 범위

본 문서는 WHATtodo v1.0(MVP) 기능 범위 내의 사용자 시나리오를 다룹니다:

- 회원가입 및 로그인
- 할일 생성, 수정, 완료, 삭제
- 캘린더 뷰 활용
- 알림 관리
- 휴지통 관리
- 프로필 설정

### 1.3 작성 방식

각 시나리오는 실제 사용자의 생각, 감정, 행동을 포함한 내러티브 형식으로 작성되었습니다. 시나리오는 다음 요소를 포함합니다:

- 사용자의 목표와 동기
- 구체적인 단계별 액션
- 각 단계에서의 사용자 감정 및 생각
- UI 화면과의 상호작용
- 예상되는 문제점과 해결 방안

- 거주지: 서울 강남구
- 기술 수준: 중급 (다양한 업무 도구 사용 경험 있음)

**라이프스타일:**
- 평일 오전 9시 출근, 오후 6시 퇴근 (종종 야근)
- 하루에 처리해야 할 업무가 평균 10~15개
- 회의, 이메일, 보고서 작성 등 다양한 업무 병행
- 스마트폰과 노트북을 번갈아 사용

**목표:**
- 간단하지만 명료한 일정 관리
- 중요한 업무를 놓치지 않기
- 빠른 입력과 조회로 시간 절약

**페인포인트:**
- 복잡한 할일 관리 도구는 오히려 시간 낭비
- 여러 도구를 사용하다 보니 업무가 분산됨
- 긴급/중요 업무를 놓치는 경우가 종종 발생
- 과거 사용했던 앱들이 너무 무겁고 느림

**사용 패턴:**
- 아침 출근길에 오늘 할 일 확인 (모바일)
- 점심시간에 오후 일정 재정리
- 퇴근 전 미완료 업무 체크 및 내일로 이동
- 하루 총 사용 시간: 10~30분

**기대효과:**
- 빠짐없는 업무 처리로 상사의 신뢰 획득
- 퇴근 시간 단축 및 업무 효율 향상
- 스트레스 감소

- 주말에 다음 주 일정을 캘린더로 확인
- 하루 총 사용 시간: 5~15분

**기대효과:**
- 과제 마감일 준수로 학점 향상

- 여유로운 대학 생활

## 3. 주요 사용자 시나리오

### 3.1 시나리오 SC-001: 신규 사용자 온보딩 여정

**시나리오 ID:** SC-001
**시나리오 제목:** 첫 방문 사용자의 회원가입부터 첫 할일 완료까지
**관련 페르소나:** 대학생 김은호
**사용자 목표:** 앱에 가입하고 첫 할일을 등록하여 완료 처리하기
**사전 조건:**
- WHATtodo 앱에 접속한 적 없음
- 친구의 추천으로 앱에 대해 들음
- 스마트폰 사용 중

---

#### 시나리오 스토리

**시간:** 월요일 오전 10시, 컴퓨터공학 개론 수업 중

교수님이 "이번 주 금요일까지 자료구조 과제 제출하세요"라고 말씀하신다. 은호는 '아, 또 과제 마감일 놓치면 안 되는데...'라고 생각하며 자리로 돌아온다. 친구가 추천했던 WHATtodo 앱이 떠오른다.

"일단 한번 써보자."

은호는 스마트폰으로 WHATtodo 웹사이트에 접속한다. 깔끔한 랜딩 페이지가 보이고, 화면 중앙에 "시작하기" 버튼이 크게 보인다.

"오, 디자인 깔끔하네."

버튼을 누르자 회원가입 화면이 나타난다. 이메일, 비밀번호, 닉네임 입력란이 간단하게 배치되어 있다.

**입력 내용:**
- 이메일: eunho.kim@student.ac.kr
- 비밀번호: ••••••••••
- 닉네임: 은호

"복잡한 정보 안 물어봐서 좋네. 다른 앱들은 이것저것 다 물어보던데."

입력을 마치고 "가입하기" 버튼을 누른다. 잠깐의 로딩 후 자동으로 로그인되며 대시보드 화면으로 이동한다.

**대시보드 첫 화면:**
- 상단: "환영합니다, 은호님!" 메시지
- 중앙: 빈 화면에 "첫 할일을 추가해보세요" 안내 문구
- 하단: "+" 버튼이 눈에 띄게 배치

"오케이, 바로 과제 등록해야지."

은호는 "+" 버튼을 터치한다. 할일 생성 모달이 부드럽게 슬라이드 업 된다.

**할일 입력:**
- 제목: 자료구조 과제 제출
- 설명: 스택과 큐 구현 코드 작성
- 우선순위: 높음 (빨간색으로 표시됨)
- 마감일: 2025-11-29 (금요일)
- 메모: 교수님 메일로 제출

"마감일 설정하니까 알림도 온다고 하네. 좋아!"

"저장" 버튼을 누른다. 모달이 닫히며 대시보드에 방금 만든 할일 카드가 나타난다.

**할일 카드 모습:**
```
[📌 높음] 자료구조 과제 제출
D-4 | 11월 29일 금요일
□ 미완료
```

"오, 진짜 깔끔하다. D-4까지 보여주네."

은호는 만족스러워하며 화면을 아래로 스크롤한다. 하단에 "완료된 할일 보기"와 "휴지통" 메뉴가 보인다.

"일단 과제 끝내고 완료 처리해봐야겠다."

---

**3일 후, 목요일 오후 11시 (과제 완료 후)**

은호는 밤늦게 과제를 완성했다. 피곤하지만 뿌듯한 마음에 WHATtodo를 켠다.

"드디어 끝났다! 체크 해야지."

할일 카드의 체크박스를 터치한다. 부드러운 애니메이션과 함께 카드가 회색으로 변하며 체크 표시가 된다.

**완료된 카드 모습:**
```
[✓ 완료] 자료구조 과제 제출
완료일: 11월 28일 목요일
```

"뭔가 보람차네. 이거 계속 써봐야겠다."

#### UI/UX 고려사항

**1. 회원가입 화면:**
- 최소한의 정보만 요구 (이메일, 비밀번호, 닉네임)
- 실시간 유효성 검증 (이메일 형식, 비밀번호 강도)
- 에러 메시지는 친절하고 구체적으로 ("이미 사용 중인 이메일입니다")

**2. 대시보드 (첫 방문 시):**
- 빈 화면에 명확한 CTA ("첫 할일을 추가해보세요")
- "+" 버튼은 화면 하단 중앙에 고정 (Floating Action Button)
- 환영 메시지로 사용자 이름 표시

**3. 할일 생성 모달:**
- 슬라이드 업 애니메이션으로 부드러운 전환
- 필수 항목(제목)과 선택 항목 명확히 구분
- 날짜 피커는 캘린더 형식으로 제공
- 우선순위는 색상으로 시각화 (높음: 빨강, 보통: 파랑, 낮음: 회색)

**4. 완료 처리:**
- 체크박스 터치 시 부드러운 애니메이션
- 완료된 카드는 시각적으로 구분 (회색 처리, 줄 긋기)
- 완료 시 마이크로 인터랙션 (체크 아이콘 애니메이션)

**5. 피드백:**
- 모든 액션에 즉각적인 피드백 제공
- 성공 메시지는 토스트 형식으로 간단히 표시
- 로딩 시간이 긴 경우 스켈레톤 UI 표시

---

#### 잠재적 문제점 및 해결 방안

**문제 1: 회원가입 시 이메일 중복**
- 문제: 이미 사용 중인 이메일로 가입 시도
- 해결: "이미 사용 중인 이메일입니다. 로그인하시겠어요?" 메시지와 함께 로그인 페이지 링크 제공

**문제 2: 첫 할일 생성 시 혼란**
- 문제: 너무 많은 입력 필드로 인한 부담
- 해결: 제목만 필수로 하고, 나머지는 선택사항임을 명확히 표시. "나중에 수정 가능합니다" 안내

**문제 3: 완료 처리 실수**
- 문제: 실수로 체크박스를 눌러 완료 처리
- 해결: 완료된 할일의 체크박스를 다시 터치하면 완료 취소 가능 (토글 방식)

**문제 4: 마감일 설정 어려움**
- 문제: 날짜 입력 방식이 불편
- 해결: 직관적인 캘린더 UI 제공, "오늘", "내일", "이번 주" 등 빠른 선택 옵션 제공

### 3.2 시나리오 SC-002: 직장인의 하루 일정 관리

**시나리오 ID:** SC-002
**시나리오 제목:** 출근길 일정 확인 및 업무 우선순위 조정
**관련 페르소나:** 직장인 김철수
**사용자 목표:** 오늘 해야 할 업무를 확인하고 우선순위에 따라 효율적으로 처리하기
**사전 조건:**
- WHATtodo 계정 보유 및 로그인 상태
- 이미 여러 개의 할일이 등록되어 있음

#### UI/UX 고려사항

**1. 오늘 할일 자동 표시:**
- 앱 실행 시 기본 화면을 오늘 날짜로 설정
- 날짜 헤더에 요일까지 명확히 표시
- 우선순위 순으로 자동 정렬 (HIGH > NORMAL > LOW)

**2. 할일 재정렬 (Drag & Drop):**
- 카드 길게 누르면 떠오르는 애니메이션
- 드래그 중 다른 카드들이 자동으로 밀림
- 놓는 위치에 따라 순서 자동 저장
- 햅틱 피드백 제공 (모바일)

**3. 모바일-웹 동기화:**
- 실시간 동기화 (1초 이내)
- 동기화 상태 표시 (아이콘)
- 오프라인 시 로컬 저장 후 자동 동기화

**4. D-day 표시:**
- 마감일까지 남은 일수 명확히 표시
- D-day: 빨간색, D-1: 주황색, D-2 이상: 회색
- 마감일 지남: "지연됨" 표시 (진한 빨간색)

**5. 완료 애니메이션:**
- 체크 시 부드러운 애니메이션
- 완료된 카드는 하단으로 이동
- 토스트 메시지로 성취감 강화

**6. 완료된 할일 조회:**
- 별도 탭으로 분리
- 완료일 순으로 정렬
- 완료 취소 기능 제공

#### 잠재적 문제점 및 해결 방안

**문제 1: 할일이 너무 많아 스크롤이 길어짐**
- 문제: 20개 이상 할일 시 스크롤 피로
- 해결: 무한 스크롤 또는 페이지네이션 적용, "오늘", "이번 주", "전체" 탭 분리

**문제 2: 드래그 앤 드롭이 의도치 않게 작동**
- 문제: 카드 클릭 시도가 드래그로 인식
- 해결: 0.5초 이상 길게 누를 때만 드래그 모드 활성화

**문제 3: 마감일 수정 시 알림 처리 혼란**
- 문제: 마감일 변경 시 기존 알림이 어떻게 되는지 불명확
- 해결: 마감일 변경 시 "알림이 재설정됩니다" 안내 메시지 표시

**문제 4: 완료된 할일과 미완료 할일 구분 어려움**
- 문제: 완료된 할일이 목록에 섞여 있음
- 해결: 완료된 할일은 하단에 별도 섹션으로 분리, 접기/펼치기 기능 제공

### 3.3 시나리오 SC-003: 대학생의 과제 마감일 관리

**사전 조건:**
- WHATtodo 계정 보유 및 로그인 상태
- 알림 설정 ON 상태
- 4개 과목 수강 중 (각 과목마다 과제 있음)

#### 시나리오 스토리

**시간:** 월요일 오후 6시, 마지막 수업 종료 후

은호는 오늘 하루 동안 교수님들로부터 총 4개의 과제를 받았다. 머릿속이 복잡하다.

집으로 가는 버스 안에서 WHATtodo를 켜고 오늘 들은 모든 과제를 입력하기로 한다.

- 우선순위: 높음

- 메모: 퀵소트, 머지소트 Python 코드

**문제 4: 과거 마감일 입력**

- 해결: "마감일이 과거입니다. 계속하시겠습니까?" 경고 메시지 표시

#### 시나리오 스토리

**시간:** 화요일 오후 3시, 사무실

철수는 할일 목록을 정리하다가 완료된 할일들을 삭제하고 있다.

"이건 끝났으니 삭제, 이것도 삭제..."

빠르게 체크하며 삭제 버튼을 누르던 중, 실수로 "Q4 실적 보고서 작성"이라는 중요한 할일을 삭제해버렸다.

"어? 잠깐, 저거 아직 안 끝났는데!"

당황한 철수는 화면을 위아래로 스크롤하지만 해당 할일이 보이지 않는다.

"아... 삭제해버렸네. 다시 만들어야 하나? 마감일이 언제였더라..."

---

불안한 마음에 하단 메뉴를 살펴보다가 "휴지통" 아이콘을 발견한다.

"휴지통? 혹시 여기 있나?"

휴지통 아이콘을 터치하자 삭제된 할일 목록 화면이 나타난다.

**휴지통 화면:**
```
🗑️ 휴지통
삭제된 할일은 30일간 보관됩니다.

[📌 높음] Q4 실적 보고서 작성
삭제일: 11월 26일 오후 3시 15분
복원 | 영구삭제

[📘 보통] 주간 회의록 작성
삭제일: 11월 25일 오전 10시
복원 | 영구삭제

[📗 낮음] 점심 메뉴 정하기
삭제일: 11월 24일
복원 | 영구삭제
```

"오! 여기 있네! 휴지통 기능이 있었구나."

철수는 "Q4 실적 보고서 작성" 카드의 "복원" 버튼을 클릭한다.

---

**복원 확인 다이얼로그:**
```
이 할일을 복원하시겠습니까?

[Q4 실적 보고서 작성]
복원 시 활성 할일 목록으로 돌아갑니다.

취소    복원
```

"당연하지!"

"복원" 버튼을 누른다. 잠깐의 로딩 후 화면 상단에 토스트 메시지가 나타난다.

**토스트 메시지:**
```
✅ 할일이 복원되었습니다.
```

휴지통 화면에서 해당 할일이 사라진다.

"휴, 다행이다."

---

철수는 하단 메뉴의 "홈" 아이콘을 터치하여 대시보드로 돌아간다.

**대시보드:**
```
[📌 높음] Q4 실적 보고서 작성 ✨ (복원됨)
D-3 | 11월 29일 금요일
□ 미완료
```

"복원된 거 표시까지 해주네. 좋아!"

할일 카드를 터치하여 상세 화면을 확인한다. 마감일, 메모 등 모든 정보가 그대로 남아있다.

"다 살아있네. 완벽!"

**10분 후**

철수는 휴지통에 있는 다른 할일들도 정리하기로 한다. "점심 메뉴 정하기"는 이미 오래된 일이라 영구 삭제하기로 한다.

#### UI/UX 고려사항

**1. 캘린더 네비게이션:**
- 월간/주간/일간 탭을 상단에 명확히 배치
- "< 이전", "다음 >" 버튼으로 주차/월 이동
- "오늘로 이동" 버튼 제공

**2. 월간 뷰:**
- 할일이 있는 날짜에 점(●) 표시
- 점 개수로 할일 수량 표시 (최대 3개)
- 공휴일은 다른 색상으로 강조 (빨간색)
- 날짜 터치 시 일간 뷰로 전환

**3. 주간 뷰:**
- 일주일을 세로로 나열
- 각 날짜 아래에 할일 카드 표시
- 할일이 없는 날은 빈 칸 또는 "일정 없음" 표시
- 오늘 날짜는 배경색으로 강조

**4. 일간 뷰:**
- 시간대별로 구분 (오전/오후 또는 시간 단위)
- 마감 시간이 있는 할일은 해당 시간에 배치
- 마감 시간 없는 할일은 "종일" 섹션에 표시
- 완료된 할일도 별도 섹션에 표시

**5. 공휴일 표시:**
- 공휴일 날짜에 플래그 아이콘 (🎌) 표시
- 공휴일 이름 표시 (예: "크리스마스")
- 공휴일은 빨간색으로 강조

**6. 할일 카드 인터랙션:**
- 카드 터치 시 상세 팝업
- 카드를 길게 누르면 퀵 액션 (완료, 수정, 삭제)
- 드래그로 다른 날짜로 이동 (향후 기능)

#### 잠재적 문제점 및 해결 방안

**문제 1: 한 날짜에 할일이 너무 많음**
- 문제: 주간 뷰에서 한 날짜에 5개 이상 할일 시 화면 복잡
- 해결: 최대 3개까지만 표시하고 "+ 2개 더" 표시, 터치 시 전체 목록 팝업

**문제 2: 월간 뷰에서 세부 정보 부족**
- 문제: 월간 뷰에서는 점만 표시되어 무슨 일인지 알 수 없음
- 해결: 날짜 터치 시 툴팁으로 할일 제목 간략히 표시, 또는 일간 뷰로 자동 전환

**문제 3: 공휴일 데이터 부족**
- 문제: 특정 국가/지역의 공휴일이 표시되지 않음
- 해결: 외부 API 연동 (Google Calendar, 공공 데이터 포털), 설정에서 국가 선택 옵션

**문제 4: 캘린더와 목록 뷰 간 일관성 부족**
- 문제: 캘린더에서 본 할일이 대시보드에서 다르게 보임
- 해결: 동일한 카드 디자인 유지, 색상/아이콘 일관성 유지

#### UI/UX 고려사항

**1. 설정 화면 구조:**
- 카테고리별로 명확히 구분 (프로필, 알림, 앱 정보, 계정)
- 각 섹션 사이에 구분선 또는 여백
- 중요한 액션(로그아웃)은 하단에 배치

**2. 알림 토글:**
- ON/OFF 상태가 시각적으로 명확 (색상, 아이콘)
- 토글 시 부드러운 애니메이션
- 토스트 메시지로 상태 변경 확인

**3. 프로필 수정:**
- 별도 화면으로 분리
- 프로필 사진 미리보기 제공
- 이메일은 변경 불가 (회색 처리 또는 비활성화)
- 닉네임 실시간 글자 수 표시 (2~20자)

**4. 저장 확인:**
- 저장 성공 시 토스트 메시지
- 실패 시 에러 메시지와 재시도 옵션
- 변경 사항 있을 때만 "저장" 버튼 활성화

**5. 로그아웃:**
- 확인 다이얼로그 표시 ("정말 로그아웃하시겠습니까?")
- 로그아웃 시 로그인 페이지로 이동
- 로컬 데이터는 유지 (다음 로그인 시 동기화)

#### 잠재적 문제점 및 해결 방안

**문제 1: 알림을 끈 채로 잊어버림**
- 문제: 알림을 OFF 상태로 두고 과제 마감일을 놓침
- 해결: 알림이 OFF 상태면 대시보드 상단에 "알림이 꺼져 있습니다" 배너 표시

**문제 2: 프로필 사진 업로드 실패**
- 문제: 이미지 파일이 너무 크거나 형식이 맞지 않음
- 해결: 업로드 전 파일 크기/형식 검증, 에러 메시지 표시 ("5MB 이하 JPG, PNG만 가능")

**문제 3: 닉네임 중복**
- 문제: 다른 사용자가 이미 사용 중인 닉네임

---

#### 인사이트 3: 성취감이 리텐션의 핵심
**발견:**
- 할일 완료 시 느끼는 성취감이 재사용 동기 부여
- 완료된 할일을 시각적으로 확인할 때 만족도 상승

**권장사항:**
- 완료 시 애니메이션과 토스트 메시지로 성취감 강화
- 일간/주간/월간 완료 통계 제공 (향후 기능)
- 연속 완료 일수 "스트릭" 기능 고려 (향후 기능)

---

#### 인사이트 4: 알림은 양날의 검
**발견:**
- 적절한 알림은 마감일 관리에 큰 도움
- 과도한 알림은 피로감과 알림 OFF로 이어짐

**권장사항:**
- 기본 알림은 D-1, D-day만
- 알림 시간 커스터마이징 기능 (향후)
- "조용한 시간" 설정 기능 (향후)

---

#### 인사이트 5: 실수에 관대한 UI
**발견:**
- 사용자들은 자주 실수함 (삭제, 잘못된 날짜 입력 등)
- 실수 복구가 어려우면 신뢰도 하락

**권장사항:**
- 휴지통 기능 필수 (30일 보관)
- 완료 취소 가능 (토글 방식)
- 중요한 액션(영구 삭제)은 확인 다이얼로그 표시

### 6.2 기능 우선순위 권장사항

#### Phase 1 (MVP - v1.0):

### 6.3 UI/UX 디자인 원칙

#### 원칙 1: 간결함 (Simplicity)
- 필수 정보만 표시
- 여백을 적극 활용
- 계층 구조 명확히

#### 원칙 2: 직관성 (Intuitiveness)
- 첫 사용자도 설명 없이 사용 가능
- 일반적인 UI 패턴 준수 (체크박스, 드래그 등)
- 명확한 아이콘과 레이블

#### 원칙 3: 피드백 (Feedback)
- 모든 액션에 즉각적인 피드백
- 성공/실패 명확히 구분
- 로딩 상태 표시

#### 원칙 4: 일관성 (Consistency)
- 동일한 색상 체계
- 동일한 버튼 스타일
- 동일한 애니메이션 패턴

#### 원칙 5: 접근성 (Accessibility)
- 충분한 색상 대비
- 터치 영역 최소 44x44px (모바일)
- 키보드 네비게이션 지원 (웹)

### 6.4 주요 메트릭 및 측정 방법

#### 사용자 참여도:
- **할일 완료율:** (완료된 할일 수 / 생성된 할일 수) × 100
- **목표:** 50% 이상
- **측정:** 일간/주간/월간 집계

#### 사용자 리텐션:
- **D7 리텐션:** 가입 후 7일 내 재방문율
- **목표:** 30% 이상
- **D30 리텐션:** 가입 후 30일 내 재방문율
- **목표:** 20% 이상

#### 기능 사용률:
- **캘린더 사용률:** 전체 사용자 중 캘린더 뷰 사용 비율
- **목표:** 40% 이상
- **알림 활성화율:** 알림 ON 상태 사용자 비율
- **목표:** 70% 이상

#### 기술적 성능:
- **API 응답 시간:** 평균 응답 시간
- **목표:** 1000ms 이내
- **에러율:** 전체 요청 중 에러 발생 비율
- **목표:** 1% 미만
